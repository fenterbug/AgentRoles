- **[VALIDATION.md](VALIDATION.md)**: Rules and guidelines for validating role specifications

## Overview

The Subagent Role Specification defines a standard format for creating structured, reusable definitions of AI agent personas and capabilities. This enables:

- **Consistency**: Standardized format across all role definitions
- **Reusability**: Roles can be shared and reused across projects
- **Composability**: Roles can work together or build on each other
- **Clarity**: Well-defined boundaries and responsibilities
- **Validation**: Clear criteria for well-formed specifications

## Key Concepts

### Role
A structured definition that specifies how an AI agent should behave when performing a particular type of task. Includes persona, approach, capabilities, and boundaries.

### Category
High-level grouping of related roles (technical, creative, workflow, specialized).

### Capability
A specific skill or ability that a role provides. Capabilities are discrete, testable, and composable.

### Expertise Level
The depth and sophistication at which a role operates (beginner, intermediate, advanced, expert).

## Design Goals

1. **Simple to Create**: Easy to write new role specifications
2. **Clear to Understand**: Unambiguous definitions and examples
3. **Practical to Use**: Focus on real-world application patterns
4. **Flexible Yet Structured**: Balance between freedom and consistency
5. **Human and Machine Readable**: Both developers and AI can understand and apply
